# Glossary

[← Back to Index](./00_INDEX.md)

## Purpose

This glossary defines terms, acronyms, and domain language used throughout CloudVault documentation and codebase.

## Core Concepts

### CloudVault

The name of this secure file sharing and storage application. Used as the product name throughout documentation.

**Evidence**: [replit.md](../../replit.md):1

### Object Storage

A cloud storage service that stores file data as "objects" (blobs) identified by unique keys. CloudVault uses Google Cloud Storage as its object storage backend.

**Not the same as**: Block storage (e.g., hard drives) or file systems (e.g., ext4, NTFS)

**Evidence**: [server/replit_integrations/object_storage/](../../server/replit_integrations/object_storage/)

### Share Link

A token-based URL that allows temporary, optionally password-protected access to a file without requiring authentication.

**Format**: `https://app.example.com/share/{token}`

**Database table**: `share_links`

**Evidence**: [shared/schema.ts](../../shared/schema.ts):60-79

### Presigned URL

A time-limited URL with embedded credentials that grants temporary access to upload or download a file from object storage. Generated by the server and used by the client to bypass the server for large file transfers.

**Lifetime**: Typically 15-60 minutes

**Example use**: Client uploads directly to Google Cloud Storage without sending file data through the server.

**Evidence**: [server/replit_integrations/object_storage/service.ts](../../server/replit_integrations/object_storage/service.ts)

### Object Path

The unique identifier (key) for a file blob in object storage. Format: `user_id/file_id/filename`.

**Example**: `user_abc123/file_xyz789/document.pdf`

**Evidence**: [shared/schema.ts](../../shared/schema.ts):43 (files.objectPath column)

## Technologies

### Drizzle ORM

A TypeScript ORM (Object-Relational Mapping) library that provides type-safe database queries and automatic TypeScript type inference from database schemas.

**Key feature**: Schema defined in TypeScript, types inferred automatically.

**Evidence**: [shared/schema.ts](../../shared/schema.ts), [server/db.ts](../../server/db.ts)

**Official docs**: https://orm.drizzle.team/

### shadcn/ui

A collection of reusable React components built on Radix UI primitives. Components are copied into the codebase (not installed as npm package) for full customization.

**Location**: `client/src/components/ui/`

**Evidence**: [components.json](../../components.json) (shadcn CLI config)

**Official docs**: https://ui.shadcn.com/

### TanStack React Query

A React library for fetching, caching, and synchronizing server state. Previously known as "React Query".

**Key feature**: Automatic background refetching and cache invalidation.

**Used in**: All client-side API calls

**Evidence**: [client/src/lib/queryClient.ts](../../client/src/lib/queryClient.ts)

**Official docs**: https://tanstack.com/query/

### Uppy

A JavaScript file upload library with support for drag-and-drop, progress tracking, and direct uploads to cloud storage via presigned URLs.

**Used in**: File upload flow

**Evidence**: [client/src/components/FileUpload.tsx](../../client/src/components/FileUpload.tsx) (if exists)

**Official docs**: https://uppy.io/

### Wouter

A minimalist React routing library (alternative to React Router). Lightweight (~1.5KB) with hooks-based API.

**Used in**: Client-side routing

**Evidence**: [client/src/App.tsx](../../client/src/App.tsx):11

**Official docs**: https://github.com/molefrog/wouter

### Passport.js

Node.js authentication middleware supporting multiple strategies (local, OAuth, OIDC, etc.).

**Used in**: Replit Auth integration

**Evidence**: [server/replit_integrations/auth/setup.ts](../../server/replit_integrations/auth/setup.ts)

**Official docs**: https://www.passportjs.org/

## Authentication & Security

### OIDC (OpenID Connect)

An authentication protocol built on top of OAuth 2.0. CloudVault uses OIDC to authenticate users via Replit Auth.

**Flow**: Authorization code flow with PKCE

**Evidence**: [server/replit_integrations/auth/setup.ts](../../server/replit_integrations/auth/setup.ts)

**Spec**: https://openid.net/connect/

### Replit Auth

Replit's hosted authentication service using OpenID Connect. Provides user accounts with profile information (sub, name, email).

**Configuration**: `ISSUER_URL` environment variable

**Evidence**: [server/replit_integrations/auth/](../../server/replit_integrations/auth/)

**Docs**: https://docs.replit.com/hosting/authenticating-users

### Session Cookie

An HTTP cookie containing a session ID that identifies an authenticated user. CloudVault stores session data in PostgreSQL.

**Cookie name**: `connect.sid`

**Store**: PostgreSQL via `connect-pg-simple`

**Evidence**: [server/routes.ts](../../server/routes.ts) (express-session setup in index.ts)

### bcrypt

A password hashing algorithm designed to be slow (resistant to brute-force attacks). CloudVault uses bcryptjs (pure JavaScript implementation) to hash share link passwords.

**Cost factor**: 10 rounds

**Evidence**: [server/routes.ts](../../server/routes.ts):18

**Not used for**: User passwords (handled by Replit Auth)

## Database Terms

### Migration

A versioned change to the database schema (e.g., adding a table or column). CloudVault uses Drizzle Kit for migrations.

**Command**: `npm run db:push`

**Location**: `migrations/` directory (generated)

**Evidence**: [drizzle.config.ts](../../drizzle.config.ts)

### Schema

The structure of database tables, including columns, types, indexes, and relationships. Defined in `shared/schema.ts`.

**Evidence**: [shared/schema.ts](../../shared/schema.ts)

### Cascade Delete

When deleting a parent record (e.g., folder), automatically delete all child records (e.g., subfolders and files). Implemented in application code, not database constraints.

**Evidence**: [server/storage.ts](../../server/storage.ts):80-93 (deleteFolder method)

### Connection Pool

A set of reusable database connections maintained by the application. Drizzle/pg creates a pool automatically.

**Evidence**: [server/db.ts](../../server/db.ts)

## Frontend Terms

### Radix UI

A library of unstyled, accessible UI primitives (e.g., Dialog, Dropdown, Tooltip). shadcn/ui components are built on Radix primitives.

**Evidence**: [package.json](../../package.json):17-42 (@radix-ui/* packages)

**Official docs**: https://www.radix-ui.com/

### Tailwind CSS

A utility-first CSS framework that provides low-level utility classes (e.g., `flex`, `p-4`, `text-blue-500`).

**Config**: [tailwind.config.ts](../../tailwind.config.ts)

**Evidence**: All components use Tailwind classes

**Official docs**: https://tailwindcss.com/

### HMR (Hot Module Replacement)

A feature of Vite that updates React components in the browser without a full page reload, preserving application state.

**Enabled in**: Development mode (`npm run dev`)

**Evidence**: [vite.config.ts](../../vite.config.ts)

### React Hook

A function that lets you "hook into" React features (state, lifecycle, context) from function components. Examples: `useState`, `useEffect`, custom hooks.

**Evidence**: [client/src/hooks/](../../client/src/hooks/)

**Official docs**: https://react.dev/reference/react/hooks

## Development Terms

### Monorepo

A repository containing multiple related projects or modules. CloudVault is a monorepo with `client/`, `server/`, and `shared/` modules.

**Not the same as**: Monolithic architecture (CloudVault is also monolithic - single deployment)

**Evidence**: Repository structure

### TypeScript Path Alias

A shortcut for import paths configured in `tsconfig.json`. CloudVault uses `@/` for client code and `@shared/` for shared code.

**Example**: `import { Button } from "@/components/ui/button"` instead of `../../components/ui/button`

**Evidence**: [tsconfig.json](../../tsconfig.json) (compilerOptions.paths)

### Zod

A TypeScript-first schema validation library. CloudVault uses Zod to validate API request/response payloads.

**Key feature**: Automatic TypeScript type inference from schemas.

**Evidence**: [shared/schema.ts](../../shared/schema.ts):81-95 (insert schemas)

**Official docs**: https://zod.dev/

### tsx

A Node.js runtime wrapper that executes TypeScript files directly without a separate compilation step. Used in development.

**Command**: `tsx server/index.ts`

**Evidence**: [package.json](../../package.json):7 (dev script)

**Official docs**: https://github.com/esbuild-kit/tsx

### Vite

A fast build tool and development server for frontend projects. CloudVault uses Vite for client code bundling.

**Key feature**: Native ES modules in dev, optimized bundle in production

**Evidence**: [vite.config.ts](../../vite.config.ts)

**Official docs**: https://vitejs.dev/

## Replit-Specific Terms

### Replit

A cloud-based development platform that provides hosting, databases, and authentication services. CloudVault is designed to run on Replit.

**Evidence**: `.replit` configuration file

**Official docs**: https://docs.replit.com/

### REPL_ID

A unique identifier for a Replit application (Repl). Used to configure authentication and object storage.

**Evidence**: Environment variable in auth setup

### Sidecar Service

A Replit infrastructure component that provides access to Google Cloud Storage without exposing credentials to user code.

**Evidence**: [server/replit_integrations/object_storage/](../../server/replit_integrations/object_storage/) (communicates with sidecar)

## Folder Structure Terms

### Entry Point

The first file executed when an application starts. CloudVault has two entry points: server (`server/index.ts`) and client (`client/src/main.tsx`).

**Evidence**: [package.json](../../package.json):7-9 (dev, build, start scripts)

### Component

A reusable React UI element. CloudVault has two types: shadcn/ui components (`components/ui/`) and custom components (`components/`).

**Evidence**: [client/src/components/](../../client/src/components/)

### Route

A URL path mapped to a handler (server) or component (client).

**Server routes**: Defined in `server/routes.ts`

**Client routes**: Defined in `client/src/App.tsx`

**Evidence**: [server/routes.ts](../../server/routes.ts), [client/src/App.tsx](../../client/src/App.tsx)
